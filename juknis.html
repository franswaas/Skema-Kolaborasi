Bagian 1

Pendahuluan & Tujuan Aplikasi

Skema Kolaborasi adalah aplikasi berbasis web yang dirancang untuk membantu peneliti, akademisi, perencana kota, dan praktisi pembangunan dalam memetakan, menganalisis, dan memvisualisasikan hubungan kolaboratif antar-stakeholder dalam suatu proyek atau program.

Tujuan Utama

  • Memetakan stakeholder (SH) beserta peran, kontribusi, dan manfaat yang diterima masing-masing pihak
  • Menggambarkan aliran sumber daya (finansial, tangible, intangible, barang publik, eksternalitas) antar-stakeholder
  • Memvalidasi keseimbangan dan keadilan kolaborasi melalui sistem rasio nilai
  • Menganalisis dinamika kolaborasi per fase (Pra-Kolaborasi, Operasi, Pasca)
  • Menghasilkan laporan visual dan data terstruktur untuk keperluan penelitian

Konsep Dasar: SHP dan SHF

Bagian 4

Panduan Penggunaan Fitur Utama

4.1 Menambah Stakeholder

1

Klik "+ Tambah Stakeholder"

Di panel kiri, tab SH → klik tombol di bagian bawah.

2

Isi form

Nama — nama institusi/pihak. Icon — pilih emoji atau upload gambar. Tipe — Pemerintah / Swasta / NGO / Akademisi / Komunitas / Masyarakat. Peran — SHP (dalam kotak) atau SHF (luar kotak).

3

Isi nilai kontribusi (Rp)

Nilai ini digunakan untuk menghitung proporsi dan rasio validasi. Proporsi dihitung otomatis.

4

Tentukan fase aktif

Centang Pra / Operasi / Pasca sesuai kapan stakeholder tersebut terlibat.

5

Klik Simpan

Node muncul di kanvas. Posisi bisa digeser dengan drag.

4.2 Menambah Aliran Sumber Daya

Ada dua cara membuat aliran:

Cara 1 — Tombol Hubungkan di node: Klik node → klik tombol "↔ Hubungkan" di bawah node → klik node tujuan → isi form aliran.

Cara 2 — Mode Connect: Tekan C atau klik ↔ di toolbar → klik node sumber → klik node tujuan → isi form aliran.

💡 Tip aliran dua arah Di form aliran, pilih Arah: Dua Arah (⟺) untuk menggambarkan pertukaran yang saling menguntungkan. Panah akan muncul di kedua ujung.

4.3 Menggunakan Template

1

Buka tab Template

Di panel kiri, klik tab "Template".

2

Pilih skala & template

Template tersedia dalam 4 skala: 🟢 Kecil (2-4 SH), 🟡 Menengah (5-8 SH), 🔴 Besar (8-12 SH), 🟣 Sangat Besar (12+ SH).

3

Klik "Muat Template"

Skema siap dengan node, aliran, dan nilai yang sudah terisi. Bisa langsung diedit.

4.5 Fase Kolaborasi

FaseKeterangan
Pra-KolaborasiTahap awal: penjajakan, MoU, persiapan sumber daya, penetapan peran. Aliran umumnya berupa izin, kontrak awal, dan hibah persiapan.
OperasiFase aktif: semua SH berkontribusi dan menerima manfaat. Transaksi berjalan penuh, monitoring aliran, implementasi program.
PascaEvaluasi, distribusi keuntungan akhir, laporan dampak, dan keputusan keberlanjutan kolaborasi.
ℹ Node redup Node yang tampak redup (transparan) di suatu fase berarti stakeholder tersebut tidak aktif di fase itu. Aliran juga hanya tampil sesuai fase yang dipilih.

4.6 Grup / Layer

Grup memungkinkan pengelompokan beberapa SH dalam satu kotak visual. Berguna untuk menandai sektor (Sektor Publik, Sektor Swasta, dll.) atau lokasi geografis. Buat grup melalui tab Grup di panel kiri.

4.7 Auto-Layout

Tersedia 4 pilihan layout otomatis di tab Grup:

  • ⭕ Lingkaran — node disusun melingkar, cocok untuk jaringan yang setara
  • 🏛 Hierarki — susunan bertingkat dari atas ke bawah
  • ✨ Force — simulasi gaya tarik-tolak, cocok untuk jaringan kompleks
  • ⊞ Grid — susunan kotak rapi baris-kolom
Bagian 5

Sistem Validasi & Rasio

Sistem validasi berjalan otomatis dan menampilkan badge di bagian bawah kanvas:

  • ✅ Skema Valid — semua kaidah terpenuhi
  • ⚠ N peringatan — ada kondisi yang perlu diperhatikan (tidak memblokir)
  • ❌ N error — ada pelanggaran kaidah yang perlu diperbaiki

Kaidah yang Dicek

KaidahKeterangan
Ada SHPMinimal 1 stakeholder berperan SHP (dalam kotak)
Ada SHFMinimal 1 stakeholder berperan SHF (luar kotak)
Semua SH terhubungSetiap node harus punya minimal 1 aliran
Aliran tertutupSetiap SHP harus memberi DAN menerima aliran
Label aliran terisiSemua panah harus berlabel
Nilai diterima ≥ dikeluarkanRasio nilai masuk ÷ keluar harus ≥ 1 (dikecualikan: Pemerintah & Publik/Masyarakat)
Proporsi sharing = 100%Total proporsi kontribusi semua SH harus tepat 100%
Fase terdefinisiSetiap SH harus aktif di minimal satu fase

Logika Rasio (Keseimbangan Nilai)

ℹ Pengecualian Rasio Stakeholder bertipe Pemerintah dan Masyarakat/Publik dikecualikan dari cek rasio < 1. Alasan: kedua entitas ini memang berperan memberi tanpa selalu mengharapkan imbalan finansial langsung — pemerintah menerima manfaat melalui PAD dan legitimasi, masyarakat menerima dampak sosial jangka panjang.

Cara memperbaiki error rasio:

  • Tambahkan aliran masuk (dari SH lain ke node yang error)
  • Atau kurangi nilai aliran keluar yang terlalu besar
  • Contoh realistis: tambah aliran "Pajak/retribusi" dari masyarakat ke pemerintah daerah
Bagian 6

Ekspor & Simpan Proyek

Menyimpan Proyek (💾 Simpan)

Klik tombol 💾 Simpan di header. File .json akan terunduh ke folder Downloads. File ini berisi seluruh data proyek dan bisa dibuka kembali dengan tombol 📂 Buka.

⚠ Simpan manual Tidak ada autosave — pastikan klik 💾 Simpan secara berkala agar data tidak hilang jika browser ditutup.

Membuka Proyek (📂 Buka)

Klik 📂 Buka → pilih file .json → skema otomatis dimuat ke kanvas.

Menggunakan Skenario

Tab Skenario memungkinkan simulasi "what-if" tanpa mengubah data asli. Geser slider multiplier suatu SH (misal 0.5× hingga 2×) untuk melihat bagaimana perubahan nilai kontribusi mempengaruhi rasio dan net value semua pihak.

⚠ Ketergantungan internet saat pertama kali Aplikasi memerlukan koneksi internet saat pertama dibuka untuk mengunduh font dari Google Fonts dan library html2canvas. Setelah dibuka sekali, browser biasanya meng-cache-nya. Untuk penggunaan 100% offline, font dan library bisa di-embed langsung ke dalam file HTML.

Cara Menambah Template Baru

Buka file HTML dengan text editor (Notepad++, VS Code, dsb.). Cari bagian const TEMPLATES = [. Tambahkan objek template baru dengan struktur berikut:

{ scale: 'kecil', // kecil | menengah | besar | sangat_besar scaleLabel: '🟢 Kecil', nameID: 'Nama Template (ID)', nameEN: 'Template Name (EN)', descID: 'Deskripsi singkat', icon: '🏘', nodes: [ { name: 'Nama SH', type: 'pemerintah', role: 'shp', resources: '...', interest: '...', valRp: 50000000, phases: {0:true, 1:true, 2:false}, icon: '🏛' }, // tambah node lain... ], flows: [ { fromId: 0, toId: 1, label: 'Dana hibah', type: 'financial', dir: 'one', valRp: 30000000, phases: {0:true, 1:true, 2:false} }, // tambah aliran lain... ] },
ℹ Tipe node yang valid pemerintah · swasta · ngo · akademisi · komunitas · publik
ℹ Tipe aliran yang valid financial · tangible · intangible · public · eksternalitas
ℹ Index node (fromId / toId) Index dimulai dari 0 sesuai urutan array nodes. Node pertama = 0, kedua = 1, dst.

Cara Memodifikasi Warna Tema

Semua warna didefinisikan sebagai CSS custom properties di bagian :root { ... } di awal file. Ubah nilai variabel seperti --accent, --blue, --bg, dsb. untuk mengubah tampilan keseluruhan aplikasi.

Bagian 11

Troubleshooting

MasalahKemungkinan PenyebabSolusi
Aplikasi tidak terbuka Browser terlalu lama atau file rusak Gunakan Chrome/Firefox versi terbaru. Pastikan file tidak terpotong saat transfer.
Font tidak tampil (muncul font generik) Tidak ada koneksi internet saat pertama buka Buka sekali dengan koneksi internet agar font ter-cache, atau hubungi pengembang untuk versi offline penuh.
Export gambar terpotong Bug yang sudah diperbaiki di v2.0 Pastikan menggunakan file versi terbaru. Gambar akan mencakup seluruh node.
Icon picker tidak terbuka Bug yang sudah diperbaiki di v2.0 Pastikan menggunakan file versi terbaru.
File .json tidak bisa dibuka File dibuat oleh versi berbeda atau rusak Coba buka dengan versi terbaru. File harus valid JSON dan mengandung field nodes.
Error validasi rasio terus muncul Stakeholder mengeluarkan nilai lebih besar dari yang diterima Tambahkan aliran balik yang realistis (pajak, retribusi, return investasi). Lihat Bagian 5.
Proporsi tidak 100% Floating point akumulasi Di v2.0 sudah diperbaiki otomatis dengan toleransi ±0.01%. Jika masih muncul, cek nilai Rp yang diisi.
Performa lambat dengan banyak node Browser keterbatasan memori Gunakan Chrome, tutup tab lain. Pada jaringan >15 node, sembunyikan minimap jika perlu.
